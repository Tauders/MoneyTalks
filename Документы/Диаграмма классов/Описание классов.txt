Описание классов:
Класс "Пользователь" (User) предназначен для хранения личных данных пользователей системы.
Класс "Место" (Place) предназначенен для описания места совершения транзакций.
Класс "Категория" (Category) предназначен для хранения классификации транзакций.
Класс "Счёт" (Account) предназначен для хранения информации о средствах пользователя.
Класс "Транзакция" (Transaction) предназначен для хранения транзакций, с указанием места, категории и суммы.

Описание взаимоотношений классов:
Отношение между "Пользователь" и "Счёт" необходимо для привязки одного или нескольких счетов к конкретному пользователю системы.
Отношение между "Пользователь" и "Место" необходимо для создания каждому пользователю системы уникальных мест посещения.
Отношение между "Пользователь" и "Категория" необходимо для создания каждому пользователю уникального дерева категорий.
Отношение между "Категория" и "Категория" необходимо для создания иерархической структуры категорий.
Отношение между "Место" и "Транзакция" необходимо для отображения места совершения транзакции.
Отношение между "Категория" и "Транзакция" необходимо для отображения категории совершённой транзакции.
Отношение между "Транзакция" и "Счёт" "от" (to) необходимо для отображения счёта, с которого была совершена транзакция.
Отношение между "Транзакция" и "Счёт" "куда" (from) необходимо для отображения счёта, на который была совершена транзакция.
Отношение между "Транзакция" и "Пользователь" необходимо для отображения пользователя, совершившего транзакцию.
